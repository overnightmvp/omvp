- Prompt caching reduces costs by 90% on system prompts
- Target article length: 1500-2500 words
- System prompt should specify: headline (50-60 chars), meta description (150-160 chars), 5-7 H2 sections, FAQ section with 5-7 questions
- Use streaming for better UX in future (not required for MVP)

# Transcript Cleaning Strategy
Per research, avoid regex-only cleaning. Use context-aware approach:
- Remove timestamps (e.g., [00:12] or 0:12)
- Remove repeated filler phrases ("you know", "um", "uh", "like" when not meaningful)
- Keep paragraph structure intact
- Preserve technical terms and brand names
</context>
