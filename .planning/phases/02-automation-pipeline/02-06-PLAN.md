  <name>Task 3: Create API routes for indexing and notifications</name>

  <action>
    1. Create POST /api/generation/index endpoint:
       - Validate request body: { queueItemId: string }
       - Fetch queue item and related page from generated_pages
       - Ensure page is published (published_at is not null)
       - Get page URL: `https://${creator}.${ROOT_DOMAIN}/${slug}`
       - Call requestIndexing(pageUrl)
       - Update generated_pages with indexed_at timestamp (even if MVP just logs)
       - Return JSON: { success: true, indexed: true } or { success: false, error }
       - Error handling: log errors, don't fail (indexing is best-effort)

    2. Create POST /api/generation/notify endpoint:
       - Validate request body: { queueItemId: string, status: 'success' | 'failed' }
       - Fetch queue item and user email from profiles/auth.users
       - If status === 'success':
         - Fetch page from generated_pages
         - Build page URL
         - Call sendPageReadyEmail(userEmail, { url, headline, slug })
       - If status === 'failed':
         - Extract error_message from queue item
         - Call sendPageFailedEmail(userEmail, error_message)
       - Return JSON: { success: true, emailSent: true }
       - Error handling: log Resend errors, return 200 anyway (don't block pipeline)

    3. Update orchestrator (Plan 05) to call these endpoints:
       - After status becomes 'published': call /api/generation/index
       - After indexing: call /api/generation/notify with status 'success'
       - On any failure: call /api/generation/notify with status 'failed'
  </action>
